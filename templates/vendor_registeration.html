Security & API Documentation

security Authentication

All API requests require authentication via Bearer token:

Authorization: Bearer <token>

Your token is linked to your vendor account and will expire in 24 hours.

event Session Management

Your shop status is tracked within your session:

session['vendor_is_active'] = true/false

Session timeout: 1 hour (requires re-authentication)

code API Request Examples

Update shop status:

POST /vendor/active
Content-Type: application/json
Authorization: Bearer <token>
{
"is_active": true
}

Get shop status:

GET /vendor/active
Authorization: Bearer <token>

auto_awesome API Response Examples

Success response:

HTTP/1.1 200 OK
Content-Type: application/json
{
"message": "Vendor shop is now open.",
"is_active": true
}

Error response:

HTTP/1.1 400 Bad Request
Content-Type: application/json
{
"error": "Invalid request. Missing is_active field."
}
